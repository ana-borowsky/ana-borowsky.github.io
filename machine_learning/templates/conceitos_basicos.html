Conceitos Básicos

Fundamentos essenciais para compreender o funcionamento dos algoritmos de aprendizagem de máquina.

# O que é Machine Learning?

  • Campo da Inteligência Artificial focado em aprendizado a partir de dados
  • Modelos aprendem padrões sem serem explicitamente programados
  • Baseado em estatística, otimização e teoria da computação
  • Permite previsões, classificações e tomadas de decisão

# Elementos fundamentais

  • Dados: matéria-prima do aprendizado
  • Modelo: representação matemática do problema
  • Treinamento: processo de ajuste do modelo
  • Predição: uso do modelo para gerar respostas

# Tipos de dados

> Dados estruturados

Organizados em tabelas (ex: planilhas, bancos de dados).

> Dados não estruturados

Texto, imagens, áudio, vídeo.


# Processo de treinamento

  • Divisão dos dados em treino e teste
  • Aprendizado a partir dos dados de treino
  • Avaliação com dados não vistos
  • Ajuste de parâmetros (tuning)

# Problemas comuns

> Overfitting

Modelo aprende demais os dados de treino e perde generalização.

> Underfitting

Modelo simples demais, não captura padrões relevantes.


# Features (Atributos)

  • Variáveis usadas como entrada do modelo
  • Qualidade das features impacta diretamente o desempenho
  • Engenharia de atributos é etapa crítica

# Labels (Rótulos)

  • Resposta esperada (target)
  • Usado em aprendizado supervisionado
  • Exemplo: "spam" ou "não spam"

# Pipeline de Machine Learning

  • Coleta de dados
  • Pré-processamento
  • Treinamento do modelo
  • Avaliação
  • Deploy (produção)

# No caso de um classificador

As medidas mais utilizadas são acurácia (taxa geral de acerto) e matriz de confusão. Contudo, para problemas desbalanceados, ou seja, onde há um desiquilíbrio na quantidade de exemplos por classe, recomenda-se medir precisão, revocação e f-measure (f1-score).

Por exemplo: considere um problema de duas classes (indivíduo tem ou não tem câncer). Pode acontecer da acurácia ser alta, porém em situação de desbalanceamento, o sistema estar classificando muito bem uma classe e outra não. O que pode ser um problema muito grave.

A matriz de confusão apresentada considera um problema binário, contudo no caso de múltiplas classes (W), podemos usar uma matriz WxW. Você tem na diagonal principal de uma matriz de confusão os acertos, enquanto nas demais posições confusões entre classes (erros).

Para estimar (VP, VN, FP e FN) em problemas de múltiplas classes podemos considerar uma estratégia um contra todos, na qual cada classe seria em dado momento considerada (+) e as demais como (-).


> Precisão (precision)

Quantos positivos previstos estão corretos.

Tenta responder a seguinte questão “qual a proporção de classificações positivas estão corretas?”. Um modelo que não produz falso positivo (FP) tem precisão igual a 1.

> Revocação (recall)

Quantos positivos reais foram encontrados.

Tenta responder a seguinte questão: “qual a proporção das amostras positivas foram classificadas corretamente?” Um modelo que não produza falso negativo (FN) tem revocação igual a 1.

> F-mesure (f1-score)

Consiste na média harmônica entre precisão e revocação, variando de 0 a 1 (sendo 1 perfeito).

> Acurácia (taxa de acerto)

Percentual de acertos do modelo.


# No caso de um regressor

Coeficiente de correlação (r), varia de -1 a 1 (-1: forte correlação negativa; 0: não existe correlação; 1: forte correlação positiva).

Coeficiente de determinação (r2): varia entre 0 e 1, indicando o quanto o modelo consegue explicar os valores observados. quanto maior o r2, mais explicativo é o modelo com relação à amostra.


# No caso de agrupamento

Temos a variância intra e inter-grupos (clusters) e a medida da silhueta (varia entre -1 e 1).

  • 1: indica ponto interno ao cluster e longe da borda (fronteira) entre grupos (clusters)
  • 0: indica ponto na fronteira entre grupos ou bem próximo dela.
  • -1: indica ponto associado a cluster errado.
