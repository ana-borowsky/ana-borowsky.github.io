Tarefas supervisionadas (classificação/ regressão)

Demanda dados rotulados.

Há diferentes técnicas na AM – diferem na teoria utilizada e modelos gerados.


# KNN - K-Nearest Neighbors

Principais características:

  • Baseado na similaridade entre instâncias
  • Demanda uma métrica de similaridade (algoritmo original usa distância Euclidiana)
  • Não gera um modelo explícito (não há um algoritmo de treinamento)
  • Lento na fase operacional (fase de testes)

# Árvore de decisão

Principais características:

  • Algoritmo de treinamento constrói a árvore avaliando a relevância de cada atributo no processo de classificação ou regressão.
  • Atributo mais importante está na raiz da árvore.
  • Nas folhas da árvore temos as classes.
  • Cada caminho da raiz até uma folha define uma regra do tipo (IF-THEN).
  • Exemplo típico de modelo caixa branca.
  • Técnicas de poda são utilizadas para evitar overtraining.

# Classificador Naïve Bayes

Principais características:

  • Tem como base o Teorema de Bayes
  • Usando na geração de classificadores
  • Algoritmo de treinamento calcula a probabilidade a priori de cada classe (P(cj)) e a probabilidade condicional de cada atributo de entrada ai dada cada classe cj.
  • Naïve = ingênuo, pois assume que os atributos de entrada são independentes.
  • Simples e rápido no treinamento e teste.

# SVM (Support Vector Machine)

Principais características:

  • Tem como base o Teorema de Bayes
  • Classificador binário, problemas de múltiplas classes demandam a combinação de SVMs (estratégias utilizadas: um-contra-todos ou todos-contra-todos).
  • Problemas não linearmente separáveis demandam o uso de um Kernel para transformar os dados de treinamento (gera um novo espaço, aumentando a dimensão).
  • Parâmetro de relaxamento C e Kernel devem ser ajustados simultaneamente (usa-se o algoritmo de busca: GridSearch())

# Redes Neurais Artificiais

Principais características:

  • Necessário definir a topologia da rede:
    • Camada de entrada = quantidade de atributos de entrada do problema.
    • Camada de saída = quantidade de classes.
    • Camadas ocultas definidas experimentalmente: quantidade de camadas e tamanho de cada uma (quantidade de neurônios).
    • Cada neurônio tem internamente uma função de ativação. Na camada de saída usa-se softmax().
  • A rede aprende a partir dos dados via algoritmo que busca minimizar o erro ajustando os pesos nas conexões entre neurônios (Backprogragation).
  • Muito suscetível a overtraining (há técnicas para evitar isto).

# Avaliação de Classificadores Binários

Matriz de confusão (problema binário - duas classes):

Métricas:

  • Taxa de acerto (Acurácia) = (VP+VN) / (VP+VN+FP+FN)
  • Precisão (Precision): P = VP / (VP+FP) ⇒ Um modelo que não produz falso positivo(FP) tem precisão igual a 1. (quantos itens selecionados são relevantes)
  • Revocação (Recall): R = VP / (VP + FN) ⇒ Um modelo que não produza falso negativo (FN) tem revocação igual a 1. (quantos item relevantes foram selecionados)
  • F-measure: 2/(1/P+1/R) => média harmônica entre precisão e revocação

# Classificação: Resumo das etapas do processo.

Representação do problema

  • Conhecer muito bem o problema que se pretende modelar.
  • Entender o ambiente: estático ou dinâmico? Dados acessíveis?
  • Elaborar uma representação apropriada.
    • Definir os atributos que descrevem o problema (entrada e saída)
    • Selecionar atributos relevantes (filtragem)
  • Definir as técnicas de AM que serão utilizadas na criação dos modelos.
    • O que se espera? Alto desempenho apenas? Explicação para os resultados?
  • Definir o protocolo experimental e as métricas de avaliação a serem utilizadas.
  • Realizar experimentos para avaliar a capacidade de generalização dos modelos criados.
  • Analisar os erros do modelo (matriz de confusão), se necessário voltar na escolha da técnicas de AM ou mesmo na representação do problema.
  • Colocar o modelo selecionado em campo (treinar com a maior quantidade de dados possível).
  • Acompanhar o desempenho do modelo implantado. Ficar atento para comportamentos indesejáveis que podem ser causados pela presença de viés nos dados de treinamento.

# Treinamento e Avaliação de Classificadores e Regressores

Técnicas a serem consideradas:

  • KNN
  • Naive Bayes
  • Árvores
  • SVM
  • Redes Neurais Artificiais (MLP - MultiLayer Perceptron)

Métodos de Avaliação

  • Holdout
  • Validação Cruzada
  • Leave-One-Subject-Out (LOSO)
  • Resubstuição

Métricas

  • Classificação: acurácia, precisão, revocação, f-score, matriz de confusão
  • Regressão: coeficiente de determinação (r2) e mae (mean absolute error)
