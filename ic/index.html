Smart Metering com AIoT: Resumos, Tabelas e Termos

Visão Geral

Este artigo investiga a viabilidade de executar modelos de Machine Learning (ML) em dispositivos de baixo custo, como o microcontrolador ESP32-S3, para reconhecimento de dígitos em medidores inteligentes (smart meters). O estudo foca em converter modelos treinados em computadores potentes para um formato leve (TensorFlow Lite) e avalia o desempenho em termos de precisão, tempo de inferência e desafios no processo de conversão.

Visão Geral

O desempenho de sistemas de visão computacional depende criticamente da qualidade da imagem. Este trabalho avalia cinco câmeras de baixa resolução (OV2640, OV7670, OV7725, NT99141, GC0308) conectadas a um ESP32-S3. A análise utiliza métricas quantitativas de nitidez, contraste, ruído e tempos de operação para selecionar a câmera mais adequada para aplicações de ML embarcado.

Visão Geral

Este artigo avalia a eficácia de várias técnicas de aumento de dados (Data Augmentation) para melhorar o desempenho de modelos de ML em dispositivos com recursos limitados. Usando três conjuntos de dados de dígitos, o estudo testa o impacto de ruído, brilho, contraste e transformações geométricas na precisão do modelo, buscando encontrar um "ponto ótimo" de aumento que melhore a performance sem causar overfitting.

Principais Descobertas

  • Impacto Positivo: O aumento de dados geralmente melhora a performance do modelo.
  • Ponto de Saturação: Existe um limite a partir do qual o aumento de dados não traz mais ganhos.
  • Técnicas mais Eficazes: Ajustes de brilho, translação horizontal e deformações elásticas mostraram-se particularmente eficazes.
  • Técnicas a Evitar: Rotações com ângulos muito grandes prejudicaram a precisão.

Glossário de Termos Técnicos

Acurácia (Accuracy)

Métrica que mede a proporção de previsões corretas de um modelo em relação ao total de previsões. É a forma mais comum de avaliar o desempenho geral de um modelo de classificação.

AIoT (Artificial Intelligence of Things)

A integração de Inteligência Artificial (IA) com a Internet das Coisas (IoT). Permite que dispositivos conectados analisem dados e tomem decisões inteligentes de forma local e autônoma.

Backpropagation

Algoritmo fundamental para o treinamento de redes neurais. Ele calcula o gradiente da função de perda em relação aos pesos da rede, permitindo que os pesos sejam ajustados para minimizar o erro.

Camada Densa (Dense Layer)

Também chamada de camada totalmente conectada, é um tipo de camada em uma rede neural onde cada neurônio recebe entrada de todos os neurônios da camada anterior. É comumente usada nas partes finais de uma CNN para classificação.

CNN (Rede Neural Convolucional)

Tipo de rede neural profunda especialmente eficaz na análise de imagens, sendo a base para modelos de reconhecimento de dígitos como o LeNet-5.

Data Augmentation (Aumento de Dados)

Técnicas para criar artificialmente novos dados de treinamento (ex: rotacionando imagens) para tornar o modelo mais robusto e evitar overfitting.

Dataset (Conjunto de Dados)

Uma coleção de dados (imagens, textos, números) usada para treinar, validar e testar modelos de machine learning.

Edge AI / Computação de Borda

Processamento de algoritmos de IA diretamente no dispositivo local (na "borda" da rede), em vez de na nuvem. Reduz latência, aumenta a privacidade e economiza dados.

Época (Epoch)

Uma passagem completa de todo o conjunto de dados de treinamento através da rede neural durante o processo de treinamento.

ESP32-S3

Um microcontrolador de baixo custo e alta performance, ideal para aplicações de AIoT, com capacidade para executar modelos de ML de pequeno porte.

Função de Ativação

Uma função matemática aplicada a cada neurônio em uma rede neural que decide se ele deve ser "ativado" ou não. Exemplos incluem ReLU, Sigmoid e Tanh.

Função de Perda (Loss Function)

Uma função que mede o quão errada é a previsão do modelo em comparação com o valor real. O objetivo do treinamento é minimizar o valor desta função.

Hiperparâmetros

Configurações externas de um modelo que não são aprendidas dos dados. São definidas antes do treinamento, como a taxa de aprendizagem, o número de épocas e o tamanho do lote.

Inferência

O processo de usar um modelo de machine learning já treinado para fazer previsões sobre novos dados nunca vistos antes.

LoRaWAN

Um protocolo de rede de longa distância e baixo consumo de energia (LPWAN), ideal para conectar dispositivos IoT que enviam pequenas quantidades de dados, como os medidores inteligentes.

Matriz de Confusão

Uma tabela que resume o desempenho de um modelo de classificação, mostrando os verdadeiros positivos, falsos positivos, verdadeiros negativos e falsos negativos.

Otimizador (Optimizer)

Algoritmo que ajusta os pesos da rede neural para minimizar a função de perda. Exemplos populares são Adam, SGD e RMSprop.

Overfitting (Superajuste)

Ocorre quando um modelo de ML "decora" os dados de treinamento e perde a capacidade de generalizar para dados novos, resultando em baixo desempenho.

Quantização

Uma técnica de otimização que reduz a precisão numérica dos pesos de um modelo (ex: de 32 bits para 8 bits). Isso diminui o tamanho do modelo e acelera a inferência, sendo crucial para dispositivos de borda.

Tamanho do Lote (Batch Size)

O número de amostras de dados que são propagadas através da rede de uma só vez durante o treinamento. Um hiperparâmetro que afeta o uso de memória e a velocidade de treinamento.

Taxa de Aprendizagem (Learning Rate)

Um hiperparâmetro que controla o tamanho do passo que o otimizador dá para ajustar os pesos do modelo. É um dos hiperparâmetros mais importantes a serem ajustados.

TensorFlow Lite (TFLite)

Versão otimizada do TensorFlow, criada para implantar modelos de ML em dispositivos móveis e embarcados com recursos limitados.

Treinamento, Validação e Teste

A divisão padrão de um dataset. O conjunto de treinamento é usado para ensinar o modelo. O de validação é usado para ajustar os hiperparâmetros. O de teste é usado para avaliar o desempenho final do modelo em dados desconhecidos.
